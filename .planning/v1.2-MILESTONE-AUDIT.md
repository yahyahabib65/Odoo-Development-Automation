## Tech Debt Summary

### Phase 12: Template Correctness & Auto-Fix

1. **Auto-fix functions lack runtime dispatch** — `fix_missing_mail_thread()` and `fix_unused_imports()` are implemented with 15 passing tests but have no caller in the production code path. The `validate` CLI command's `--auto-fix` flag only triggers `run_pylint_fix_loop` (pylint codes W8113, W8111, C8116, W8150, C8107). A `run_docker_fix_loop()` function or extension to the validate CLI is needed to wire these functions into runtime execution.

2. **identify_docker_fix() never called from non-test code** — `FIXABLE_DOCKER_PATTERNS` was expanded from 4 to 5 patterns and `_DOCKER_PATTERN_KEYWORDS` includes mail.thread keywords, but no code outside tests calls `identify_docker_fix()` to detect and dispatch Docker error fixes.

### Total: 2 items across 1 phase
