  <action>
    Implement three new Docker fix functions in auto_fix.py:

    1. `fix_xml_parse_error(module_path: Path, error_output: str) -> bool`:
       - Parse the error_output to extract the filename and line number of the XML error (Odoo logs include `File "...", line N`)
       - Use a heuristic approach: detect common XML issues like mismatched tags by reading the XML file and attempting to parse with lxml/xml.etree
       - If a mismatched closing tag is found (the most common case from Odoo Docker logs), fix it
       - If the XML cannot be mechanically fixed (complex structural issues), return False
       - This is a best-effort fix -- the most common case is a typo in a closing tag name

    2. `fix_missing_acl(module_path: Path, error_output: str) -> bool`:
       - Scan models/ directory for all .py files that define `_name = "..."` to get model names
       - Check if security/ir.model.access.csv exists. If not, create it with header + one access rule per model
       - If CSV exists, parse it and add missing model entries
       - Access rule format: `access_{model_underscore},access.{model_dot},model_{model_underscore},base.group_user,1,1,1,0`
       - Create security/ directory if it does not exist
       - Also check __manifest__.py includes 'security/ir.model.access.csv' in 'data' key; if not, add it

    3. `fix_manifest_load_order(module_path: Path, error_output: str) -> bool`:
       - Read __manifest__.py and extract the 'data' list
       - Identify files that define actions (contain `ir.actions.act_window` or `<act_window`) and files that reference actions (contain `action=` attribute in menu items)
       - Reorder so definition files come before reference files
       - Write the updated data list back to __manifest__.py
       - Use AST for safe manifest parsing (same pattern as existing C8107 fix)

    4. Update `run_docker_fix_loop` dispatch dict to include all 3 new functions:
       ```python
       dispatch = {
           "xml_parse_error": fix_xml_parse_error,
           "missing_acl": fix_missing_acl,
           "manifest_load_order": fix_manifest_load_order,
           "missing_mail_thread": fix_missing_mail_thread,
       }
       ```
       Note: `missing_import` stays without a fix function -- it requires the user to install a Python package or add an Odoo module dependency, which is not mechanically deterministic. The dispatch returns False for unmatched patterns, which is correct.

    5. Update function signatures for fix functions that need error_output: `fix_xml_parse_error`, `fix_missing_acl`, and `fix_manifest_load_order` take `(module_path, error_output)`. Update `run_docker_fix_loop` to pass `error_output` to all dispatch calls. Update `fix_missing_mail_thread` call to still work (it only needs module_path).

    IMPORTANT: Use immutable read-transform-write patterns per project coding style. Do not mutate data structures in place.
