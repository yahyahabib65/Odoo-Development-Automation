Create `python/tests/test_golden_path.py` with:

1. Module-level `pytestmark = pytest.mark.docker` marker.

2. A `skip_no_docker` mark using `pytest.mark.skipif(not check_docker_available(), reason="...")`.

3. A module spec dict (`GOLDEN_PATH_SPEC`) defined as a module-level constant with:
   - `module_name: "hr_training"`
   - `depends: ["base", "mail"]` (NOT "hr" -- "hr" adds real Odoo dependency complexity; the goal is regression testing OUR templates, not Odoo's dependency resolution. "base" + "mail" are sufficient to exercise all Phase 12 fixes.)
   - Model 1 (`hr.training.course`): Char name (required), Integer duration, Text description, Float total_hours (with compute="_compute_total_hours", depends=["duration"]), Selection state (draft/confirmed/done). All fields have `string=` attributes.
   - Model 2 (`hr.training.session`): Char name (required), Date date, Integer attendee_count. All fields have `string=` attributes. No computed/onchange/constrained fields.
   - This spec exercises: needs_api=True (model 1) and needs_api=False (model 2), inherit_list with mail.thread on both models, clean manifest (no installable/auto_install), clean test imports (no ValidationError).

4. A `@pytest.fixture(scope="module")` named `rendered_module` that uses `tmp_path_factory` to render the spec once and return the module directory Path. This fixture is shared by all three tests.

5. Three test functions decorated with `@skip_no_docker` (install and run_tests also need Docker):
   - `test_golden_path_render(rendered_module)`:
     - Assert module directory exists
     - Assert expected files exist: `__manifest__.py`, `__init__.py`, `models/__init__.py`, `models/hr_training_course.py`, `models/hr_training_session.py`, `views/hr_training_course_views.xml`, `views/hr_training_course_action.xml`, `views/hr_training_session_views.xml`, `views/hr_training_session_action.xml`, `views/menu.xml`, `security/security.xml`, `security/ir.model.access.csv`, `tests/__init__.py`, `tests/test_hr_training_course.py`, `tests/test_hr_training_session.py`
     - This test does NOT need skip_no_docker since render does not use Docker. Remove the decorator for this test only.
   - `test_golden_path_docker_install(rendered_module)`:
     - Call `docker_install_module(rendered_module)`
     - Assert `result.success is True` with helpful error message including `result.error_message`
     - Assert `"ImportError" not in result.log_output`
   - `test_golden_path_docker_tests(rendered_module)`:
     - Call `docker_run_tests(rendered_module)`
     - Assert `len(results) > 0` ("Expected at least 1 test result")
     - Assert `all(r.passed for r in results)` with detailed failure message listing failed test names
     - Assert `all(r.test_name for r in results)` ("Test names must be non-empty")

6. Use pytest's `dependency` behavior via ordering: since all 3 tests share the same fixture, if render fails (fixture error), both Docker tests are automatically skipped. For install vs run_tests ordering, rely on pytest's default alphabetical ordering (docker_install before docker_tests) and document in docstrings that the tests are designed to run in order.
