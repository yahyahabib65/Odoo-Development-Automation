  <action>
    RED phase: Create python/tests/test_i18n_extractor.py with all behavior tests above. Use tempfile.TemporaryDirectory to create sample .py and .xml files for testing. Run pytest -- tests should all FAIL.

    GREEN phase: Create python/src/odoo_gen_utils/i18n_extractor.py with these functions:

    1. `extract_python_strings(file_path: Path) -> list[tuple[str, str, int]]`
       - Use `ast.parse()` to build AST from file content
       - Walk AST nodes looking for `ast.Call` where func is `ast.Name(id='_')`
       - Extract the first string argument (must be `ast.Constant` with str value)
       - Return list of (msgid, filename, line_number) tuples

    2. `extract_xml_strings(file_path: Path) -> list[tuple[str, str, int]]`
       - Use `xml.etree.ElementTree.parse()` to parse XML
       - Scan all elements for `string` attribute values
       - Also scan `<label>` elements for text content
       - Wrap in try/except for malformed XML (return empty list)
       - Return list of (msgid, filename, line_number) tuples
       - Note: ElementTree does not track line numbers reliably; use 0 as line for XML entries

    3. `extract_translatable_strings(module_path: Path) -> list[tuple[str, str, int]]`
       - Walk module_path recursively
       - Call extract_python_strings on every .py file
       - Call extract_xml_strings on every .xml file
       - Return combined, sorted list

    4. `generate_pot(module_name: str, strings: list[tuple[str, str, int]]) -> str`
       - Build POT header per CONTEXT.md Decision D format (Project-Id-Version: Odoo Server 17.0, etc.)
       - Use $TIMESTAMP = current UTC datetime formatted as "YYYY-MM-DD HH:MM+0000"
       - For each unique msgid, collect all source references (#: file:line)
       - Deduplicate msgids, merge source references
       - If strings list is empty, still produce the header (per locked decision)
       - Return the complete POT file content as a string

    REFACTOR phase: Ensure all functions use immutable patterns (no mutation of input data). Verify all edge cases pass.

    Known limitation (per CONTEXT.md Decision D): field `string=` auto-translations from Python field declarations (e.g., `fields.Char(string="My Field")`) are NOT extracted. This is documented as a known gap, acceptable for v1.
