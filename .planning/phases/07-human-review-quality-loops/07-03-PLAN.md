    GREEN phase: Create python/src/odoo_gen_utils/auto_fix.py:

    **Constants:**
    ```python
    # Pylint codes that can be mechanically auto-fixed
    FIXABLE_PYLINT_CODES: frozenset[str] = frozenset({
        "W8113",  # redundant string= parameter on field
        "W8111",  # renamed field parameter
        "C8116",  # superfluous manifest key
        "W8150",  # absolute import should be relative
        "C8107",  # missing required manifest key
    })

    MAX_FIX_CYCLES: int = 2

    # Docker error pattern IDs that can be auto-fixed
    FIXABLE_DOCKER_PATTERNS: frozenset[str] = frozenset({
        "xml_parse_error",
        "missing_acl",
        "missing_import",
        "manifest_load_order",
    })
    ```

    **Functions:**

    1. `is_fixable_pylint(violation: Violation) -> bool`
       - Return violation.rule_code in FIXABLE_PYLINT_CODES

    2. `fix_pylint_violation(violation: Violation, module_path: Path) -> bool`
       - Read the file at module_path / violation.file
       - Apply mechanical fix based on rule_code:
         - W8113: Use regex to remove `string="..."` or `string='...'` from the field definition line
         - W8111: Map known renamed parameters (e.g., `track_visibility` -> `tracking`, `oldname` -> remove) and replace
         - C8116: Parse __manifest__.py as Python dict literal, remove the superfluous key (the violation message contains the key name)
         - W8150: Replace `from odoo.addons.{module}` with relative import `from .`
         - C8107: Parse __manifest__.py and add the missing required key with empty/default value
       - Write fixed content back to file
       - Return True if fix was applied, False if not applicable
       - IMPORTANT: Use immutable patterns -- read content, create new content, write back. Never mutate in-place.

    3. `fix_pylint_violations(violations: tuple[Violation, ...], module_path: Path) -> tuple[int, tuple[Violation, ...]]`
       - Filter to fixable violations only
       - Apply fix_pylint_violation for each
       - Count successes
       - Return (fixed_count, remaining_violations) where remaining = non-fixable + failed fixes

    4. `run_pylint_fix_loop(module_path: Path, pylintrc_path: Path | None = None) -> tuple[int, tuple[Violation, ...]]`
       - Cycle 1: run_pylint_odoo -> fix_pylint_violations -> count
       - Cycle 2: run_pylint_odoo again -> fix_pylint_violations -> count
       - After 2 cycles, return (total_fixed, remaining)
       - If cycle 1 produces 0 fixable violations, skip cycle 2

    5. `identify_docker_fix(diagnosis: str) -> str | None`
       - Match diagnosis text against FIXABLE_DOCKER_PATTERNS
       - Return the pattern ID if fixable, None if not
       - Uses simple string matching against known pattern explanations from error_patterns.json

    6. `format_escalation(violations: tuple[Violation, ...]) -> str`
       - Group violations by file
       - For each file: list violations with line number and message
       - Include suggestion if available
       - Format per CONTEXT.md Decision E:
         ```
         Auto-fix exhausted. Remaining violations:

         [models/sale_order.py:42] E8103: Model _description missing
           -> Add `_description = "Sales Order"` to the model class
         ```
       - Return "No remaining issues." if violations is empty

    REFACTOR phase: Ensure clean separation. All file I/O in fix functions uses Path.read_text / Path.write_text. No global state.
