**workflows/scaffold.md** — The end-to-end scaffold workflow. This is the primary workflow referenced by `/odoo-gen:new` and the `odoo-scaffold` agent. Must define:

1. **Input Parsing Phase:**
   - Receive module description from `$ARGUMENTS`
   - Parse the description to infer: module technical name (convert to snake_case), module title, models with fields, dependencies
   - For technical name: convert description to snake_case module name (e.g., "inventory tracking" -> "inventory_tracking")
   - For models: identify nouns as potential models, verbs as potential actions/states
   - For fields: infer field types from context (e.g., "price" -> Float, "name" -> Char, "date" -> Date, "description" -> Text)
   - For dependencies: always include "base", add "mail" if description mentions notifications/messages, add "stock" if mentions inventory, etc.

2. **Spec Confirmation Phase:**
   - Present the inferred spec to the user in a readable format:
     - Module name and title
     - Dependencies list
     - For each model: name, description, and fields table (name, type, required, description)
   - Wait for user to confirm or request changes
   - If user requests changes, update the spec accordingly

3. **Generation Phase:**
   - Create the module spec JSON matching the `render_module` function's expected input format
   - Call `$HOME/.claude/odoo-gen/bin/odoo-gen-utils render-module --spec-file /tmp/odoo_gen_spec.json --output-dir ./<module_name>/`
   - Announce each generation step: "Generating manifest... Generating models... Generating views... Generating security... Generating tests... Generating demo data..."
   - After generation, list all created files with `find ./<module_name>/ -type f | sort`

4. **Post-Generation Phase:**
   - Print summary: "Module '<module_title>' scaffolded at ./<module_name>/"
   - Print next steps: "To install: copy to your Odoo addons directory and update the modules list"
   - Suggest running `/odoo-gen:validate ./<module_name>/` (when available in Phase 3)

Include Odoo 17.0 specifics in the workflow:
- Default odoo_version: "17.0" (read from defaults.json if available)
- Default license: "LGPL-3"
- OCA directory structure is mandatory
- One Python file per model
