**python/src/odoo_gen_utils/renderer.py** — Jinja2 rendering engine:
1. `create_renderer(template_dir: Path) -> Environment` — Creates Jinja2 Environment with:
   - `FileSystemLoader` pointing to template_dir
   - `StrictUndefined` (fail on missing variables — Pitfall 1 prevention)
   - `keep_trailing_newline=True`, `trim_blocks=True`, `lstrip_blocks=True`
   - Custom filter `model_ref`: `"inventory.item"` -> `"model_inventory_item"` (Pitfall 5 prevention)
   - Custom filter `to_class`: `"inventory.item"` -> `"InventoryItem"` (capitalize each word)
   - Custom filter `to_python_var`: `"inventory.item"` -> `"inventory_item"` (dots to underscores)
   - Custom filter `to_xml_id`: `"inventory.item"` -> `"inventory_item"` (for XML id attributes)

2. `render_template(env: Environment, template_name: str, output_path: Path, context: dict) -> Path` — Renders single template to file, creates parent directories, returns output path.

3. `render_module(spec: dict, template_dir: Path, output_dir: Path) -> list[Path]` — Renders a complete Odoo module from a spec dict. The spec dict structure:
```python
{
    "module_name": "my_module",           # technical name (snake_case)
    "module_title": "My Module",          # human-readable title
    "summary": "Short description",
    "author": "Author Name",
    "website": "https://...",
    "license": "LGPL-3",
    "category": "Uncategorized",
    "odoo_version": "17.0",
    "depends": ["base"],
    "application": True,
    "models": [
        {
            "name": "my.module.item",       # Odoo dot-notation model name
            "description": "Item",
            "fields": [
                {
                    "name": "name",
                    "type": "Char",
                    "string": "Name",
                    "required": True,
                    "help": "Item name"
                },
                # ... more fields
            ]
        }
    ]
}
```
The function iterates over the spec, rendering each template for each model, and producing the full OCA directory structure:
- `__manifest__.py` (one)
- `__init__.py` (root, models/, tests/)
- `models/<model_var>.py` (one per model)
- `views/<model_var>_views.xml` (one per model, containing form+tree+search views)
- `views/<model_var>_action.xml` (one per model)
- `views/<model_var>_menu.xml` (one per model — or a single menu.xml, use discretion)
- `security/security.xml` (groups)
- `security/ir.model.access.csv`
- `tests/test_<model_var>.py` (one per model)
- `demo/demo_data.xml`
- `static/description/index.html` (minimal — "See README.rst")
- `README.rst`

4. `get_template_dir() -> Path` — Returns the path to the bundled templates directory (relative to this module).

**python/src/odoo_gen_utils/cli.py** — Click CLI:
```python
@click.group()
@click.version_option(version="0.1.0")
def main():
    """odoo-gen-utils: Python utilities for the odoo-gen GSD extension."""
    pass
```

Three commands:
1. `render` — Render a single template:
   - `--template` (required): Template file name (e.g., `manifest.py.j2`)
   - `--output` (required): Output file path
   - `--var` (multiple): Variables in `key=value` format
   - `--var-file`: Path to JSON file with template variables
   - Loads template from bundled templates dir, merges var and var-file contexts, renders to output.

2. `list-templates` — List all available templates:
   - Scans bundled templates directory
   - Prints each template name with a brief description (from template comment or filename)

3. `render-module` — Render a complete module:
   - `--spec-file` (required): Path to JSON file with full module spec
   - `--output-dir` (required): Directory to create module in
   - Reads spec, calls `render_module()`, prints list of created files.
   - Output format: one file path per line (for easy parsing by agents)

All commands handle errors with clear messages (click.echo to stderr for errors). No hardcoded paths — derive template directory from package location.

Create 15 Jinja2 templates that produce real, installable Odoo 17.0 module content. Each template MUST use Odoo 17.0 syntax exclusively (see Pitfall 2 from research).

**CRITICAL Odoo 17.0 rules for ALL templates:**
- Use `<tree>` for list views (NOT `<list>` which is Odoo 18+)
- Use inline `invisible="expression"` and `readonly="expression"` (NOT `attrs` which is deprecated)
- Version format: `{{ odoo_version }}.1.0.0` (5-part)
- `license` is a required manifest key
- Use `<odoo>` tags (NOT `<openerp>`)
- Use `from odoo import` (NOT `from openerp import`)

**CRITICAL consistency rules (Pitfall 1 prevention):**
- All templates consume from the same spec dict structure defined in renderer.py
- Model name conversions must use the Jinja2 filters: `model_ref`, `to_class`, `to_python_var`, `to_xml_id`
- Never hardcode model name transformations — always use filters

**Templates to create:**

1. **manifest.py.j2** — `__manifest__.py` with: name, version (17.0.1.0.0), author, website, license (default LGPL-3), category, summary, depends list, data file references (security/security.xml, security/ir.model.access.csv, views/*.xml), demo file references, development_status Alpha, application flag. Use research example as base but ensure all view files are listed.

2. **model.py.j2** — Python model file. Imports: `from odoo import api, fields, models`. Conditionally import `ValidationError` if model has constraints. Class with `_name`, `_description`. Fields rendered from spec with correct field types (Char, Integer, Float, Boolean, Text, Html, Date, Datetime, Selection, Many2one, One2many, Many2many). Handle `required`, `help`, `default`, `string`, `selection` options, `comodel_name`, `inverse_name`. Add `_sql_constraints` if spec includes them.

3. **init_root.py.j2** — Root `__init__.py`: `from . import models` and `from . import tests` (for development, tests are often excluded in production but OCA includes them).
   Actually for OCA convention: `from . import models` only. Tests are auto-discovered. Correct: just import models (and wizards if they exist).

4. **init_models.py.j2** — `models/__init__.py`: `from . import <model_var>` for each model file.

5. **init_tests.py.j2** — `tests/__init__.py`: `from . import test_<model_var>` for each test file.

6. **view_form.xml.j2** — Form view XML. Structure: `<odoo><record id="view_<model_xml_id>_form" model="ir.ui.view"><field name="name"><model_name>.view.form</field><field name="model"><model_name></field><field name="arch" type="xml"><form string="..."><sheet><group>` with fields. Use `<notebook>` with `<page>` for One2many fields. Use `widget="html"` for Html fields.

7. **view_tree.xml.j2** — Tree/list view XML. Use `<tree>` tag (NOT `<list>`). Show key fields in columns. Handle `optional="show"` and `optional="hide"` attributes.

8. **view_search.xml.j2** — Search view XML. Add `<field>` for Char and Many2one fields. Add `<filter>` elements for common groupings.

9. **action.xml.j2** — Window action. `<record id="action_<model_xml_id>" model="ir.actions.act_window">` with name, res_model, view_mode (tree,form), search_view_id reference. Use `context` and `domain` if needed.

10. **menu.xml.j2** — Menu items. Create root menu (if application), submenu for each model, action reference. Use `web_icon` for root menu. Structure: root menu -> model submenus.

11. **security_group.xml.j2** — Security groups XML. Create module category. Create User group (with `implied_ids` from base.group_user). Create Manager group (with `implied_ids` from the User group). Use `<odoo>` root tag.

12. **access_csv.j2** — `ir.model.access.csv`. Header: `id,name,model_id:id,group_id:id,perm_read,perm_write,perm_create,perm_unlink`. Two rows per model: user (read/write/create, no unlink) and manager (full CRUD). Use `model_ref` filter for model_id:id. Reference groups as `<module_name>.group_<module_name>_user` and `<module_name>.group_<module_name>_manager`.
