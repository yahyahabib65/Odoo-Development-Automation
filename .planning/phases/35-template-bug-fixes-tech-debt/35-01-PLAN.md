<objective>
Fix two template bugs found in the v3.1 milestone audit:

1. CRITICAL: `view_form.xml.j2` (both 17.0 and 18.0) line 20 accesses `wizard.trigger_state` unconditionally. Archival wizards injected by `_process_production_patterns()` do NOT have `trigger_state`. When a model has both `archival: true` and a `state` Selection field, `render_models()` crashes with Jinja2 StrictUndefined error. Fix: guard the wizard button block so it only renders the `invisible` attribute when `wizard.trigger_state` exists.

2. LOW: `cron_data.xml.j2` line 13 hardcodes `<field name="doall" eval="False"/>`, ignoring the `cron.doall` value from the spec dict. Fix: render dynamically from the cron dict with a default of `False`.

Purpose: Close the last two gaps in v3.1 milestone before shipping.
Output: Fixed templates + regression tests proving both scenarios work.
</objective>

    **view_form.xml.j2 fix (BOTH 17.0 and 18.0 versions):**

    Replace lines 14-21 in both files. Currently:
    ```jinja2
    {% for wizard in wizards %}
    {% if wizard.target_model == model_name %}
                        <button name="action_open_{{ wizard.name | to_xml_id }}"
                                string="{{ wizard.button_label | default('Open ' + (wizard.name | replace('.', ' ') | title)) }}"
                                type="object"
                                class="btn-primary"
                                invisible="{{ state_field.name }} != '{{ wizard.trigger_state }}'"/>
    {% endif %}
    {% endfor %}
    ```

    Change to guard the `invisible` attribute with `wizard.get('trigger_state')`:
    ```jinja2
    {% for wizard in wizards %}
    {% if wizard.target_model == model_name %}
                        <button name="action_open_{{ wizard.name | to_xml_id }}"
                                string="{{ wizard.button_label | default('Open ' + (wizard.name | replace('.', ' ') | title)) }}"
                                type="object"
                                class="btn-primary"
    {% if wizard.get('trigger_state') %}
                                invisible="{{ state_field.name }} != '{{ wizard.trigger_state }}'"/>
    {% else %}
                                />
    {% endif %}
    {% endif %}
    {% endfor %}
    ```

    This way: wizards WITH trigger_state (regular spec wizards) get the invisible attribute; wizards WITHOUT trigger_state (archival wizards) render the button without visibility constraint.

    **cron_data.xml.j2 fix:**

    Replace line 13:
    ```xml
    <field name="doall" eval="False"/>
    ```
    With:
    ```xml
    <field name="doall" eval="{{ cron.doall | default(false) | capitalize }}"/>
    ```

    Note: Jinja2 `capitalize` filter turns `true`->`True` and `false`->`False` which matches Python eval syntax in Odoo XML. Alternatively use `{{ 'True' if cron.doall else 'False' }}` if capitalize on booleans is unreliable -- test and pick the approach that produces `eval="True"` or `eval="False"`.
