<objective>
Add automatic monetary field detection to the renderer pipeline so that Float fields with monetary name patterns (amount, fee, salary, price, cost, balance, total, etc.) are rendered as `fields.Monetary` with a `currency_id` companion field auto-injected.

Purpose: Prevents `AssertionError: unknown currency_field None` at module install time and produces more accurate Odoo code without requiring users to know about `fields.Monetary`.
Output: Updated renderer.py with detection logic, updated model.py.j2 templates (17.0 + 18.0) with Monetary branch + currency_id injection, and comprehensive tests.
</objective>

Template branch ordering in model.py.j2 (both 17.0 and 18.0):
```
Selection -> Many2one/O2M/M2M -> Char+sequence -> compute -> [NEW: Monetary] -> else (catch-all)
```
Note: Monetary branch must come BEFORE the catch-all `else`, and must handle both
plain Monetary and computed Monetary (a computed field that is also Monetary needs both
compute= and currency_field=). The compute branch currently handles all computed fields
with `fields.{{ field.type }}` so it already renders the correct type -- but it misses
`currency_field=`. The Monetary branch should therefore come BEFORE the compute branch,
OR the compute branch needs an inner conditional for Monetary. Research recommends a
dedicated Monetary branch before the compute branch that handles both cases.
