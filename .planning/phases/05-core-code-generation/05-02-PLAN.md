New context keys in _build_model_context() output (per Plan 05-01):
- computed_fields: fields with "compute" key
- onchange_fields: fields with "onchange": true
- constrained_fields: fields with "constrains" list
- sequence_fields: Char fields matching reference/ref/number/code/sequence AND required
- state_field: first state/status Selection field or None
- wizards: spec.get("wizards", [])
- has_computed: bool
- has_sequence_fields: bool
- sequence_field_names: list of names from sequence_fields

Two-pass model file contract (from RESEARCH.md):
- Pass 1 (Jinja2): Renders complete file including # TODO stubs for method bodies
- Pass 2 (odoo-model-gen): Reads full file + spec.json, rewrites ENTIRE file with real method bodies
- Agent must NOT change field declarations — only fill # TODO method bodies

FORBIDDEN in Odoo 17.0 (hard constraint for all agents):
- @api.multi, @api.one, @api.returns decorators
- attrs= attribute in XML views
- states= attribute on buttons or fields in XML
- states={"draft": [...]} parameter on field definitions
- <list> tag for list views (use <tree> in 17.0, <list> is 18+)
- SavepointCase (use TransactionCase)
- self.pool.get() or _columns = {}
- env.cr.execute() without parameterized queries

REQUIRED patterns:
- Computed methods: iterate "for rec in self:" and assign rec.field = value
- Onchange methods: assign self.field = value directly (single-record UI context)
- Constrained methods: iterate "for rec in self:", raise ValidationError (not UserError)
- statusbar_visible: comma-separated state keys from selection, NO spaces

    ```python

    def _compute_field_name(self):

    ```

    - Decorator uses exact field paths from spec field.depends array
    - Method defined AFTER the field declaration (Jinja2 already does this; preserve order)
    - Infer the computation from: field name semantics + depends fields + spec context

    For @api.onchange methods:
    ```python
    @api.onchange('field_name')
    def _onchange_field_name(self):
        if self.field_name:

    ```
    - Assign to self.field directly (NOT `for rec in self:` — onchange runs on single record UI context)
    - Method name: `_onchange_{field_name}`
    - Return None normally; return `{'warning': {'title': '...', 'message': '...'}}` for validation warnings

    For @api.constrains methods:
    ```python
    @api.constrains('field_a', 'field_b')
    def _check_field_name(self):
        for rec in self:
            if rec.field_a and rec.field_b and rec.field_a > rec.field_b:
                raise ValidationError(
                    _("Field A must be less than Field B.")
                )
    ```
    - Always iterate `for rec in self:`
    - Always raise `ValidationError` (imported from odoo.exceptions) — not UserError, not Exception
    - Clear message mentioning what constraint was violated
