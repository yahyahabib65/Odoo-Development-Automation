2. Implement `_process_performance(spec: dict[str, Any]) -> dict[str, Any]` as a pure function in `renderer.py` (place it after `_process_constraints`, around line 399). The function:

   a. For each model in spec["models"]:
      - Determine search_fields: fields where type in (Char, Many2one) AND not internal (these appear in search view per view_form.xml.j2 logic). Also include Selection fields (used as group-by).
      - Determine order_fields: parse model.get("order", "") splitting by comma, taking first token of each part. Validate each field exists in model fields; skip non-existent with no error.
      - Determine domain_fields: if model has record_rules or company_id, include those field names.
      - Compute index_fields = search_fields | order_fields | domain_fields
      - INDEXABLE_TYPES = {"Char", "Integer", "Float", "Date", "Datetime", "Boolean", "Selection", "Many2one", "Monetary"}
      - For each field: if field.name in index_fields AND field.type in INDEXABLE_TYPES -> set index=True
      - Determine visible_fields: first 6 non-One2many/Html/Text fields from view_fields (tree view), union search_fields, union order_fields
      - For computed fields (field.compute is truthy): if field.name in visible_fields -> set store=True (do not overwrite if already True)
      - Process unique_together: validate all referenced fields exist in model, generate sql_constraints list of dicts {name, definition, message}. Merge with any existing model.sql_constraints.
      - Process _order: if model.order exists and all referenced fields exist, set model_order key on model dict
      - Process TransientModel: if model.get("transient") is True, set transient_max_hours (default 1.0) and transient_max_count (default 0), honoring explicit spec values

   b. Return {**spec, "models": new_models} -- pure function, no mutation.
