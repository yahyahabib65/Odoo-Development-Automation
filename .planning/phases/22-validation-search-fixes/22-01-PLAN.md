<objective>
Fix docker_install_module to use `docker compose run --rm` instead of `docker compose exec`, eliminating the PostgreSQL serialization race condition caused by two concurrent Odoo processes.

Purpose: The current `exec` approach starts a second Odoo process inside a running container where the entrypoint server is already connected to PostgreSQL. Both processes write to the same database simultaneously, causing `psycopg2.errors.SerializationFailure`. The `docker_run_tests` function already uses the correct `run --rm` pattern (lines 226-251 of docker_runner.py) -- this plan applies the same fix to `docker_install_module`.

Output: Fixed `docker_install_module` function + updated tests proving the `run --rm` pattern.
</objective>
