<objective>
Add GitHub API rate limit handling to `build_oca_index` and extend the AST analyzer to detect `_inherit`-only model extensions.

Purpose: (1) OCA crawl iterates 200+ repos with 3-5 API calls each. Without rate limit awareness, the crawl hits 403/429 errors and fails silently. (2) The AST analyzer only captures models with `_name` assignments, missing common `_inherit`-only model extensions that add fields/methods to existing models.

Output: Rate-limited `build_oca_index` + `inherited_models` field on `ModuleAnalysis` + tests for both.
</objective>

PyGithub rate limit API:
```python

gh = Github(token)
rate = gh.get_rate_limit().rate  # Rate object
rate.remaining  # int
rate.limit      # int

```

Research recommendations:
- Check rate limit every 10 repos (get_rate_limit() costs 1 API call)
- min_remaining threshold: 10
- Catch RateLimitExceededException specifically (not generic GithubException)
- Exponential backoff: 1s, 2s, 4s (max 3 retries)
- _inherit can be string or list -- normalize to tuple[str, ...]
- Add inherited_models with default=() to avoid breaking existing callers
