GREEN phase: Create python/src/odoo_gen_utils/artifact_state.py implementing:

1. `ArtifactKind(str, Enum)` with values: MODEL="model", VIEW="view", SECURITY="security", TEST="test", MANIFEST="manifest", DATA="data"

2. `ArtifactStatus(str, Enum)` with values: PENDING="pending", GENERATED="generated", VALIDATED="validated", APPROVED="approved"

3. `VALID_TRANSITIONS` dict mapping each status to allowed next statuses:
   - pending -> [generated]
   - generated -> [validated, approved]
   - validated -> [approved]
   - approved -> [] (terminal)
   Out-of-order transitions log warning via logger.warning() but proceed (OBS-01 says "does not block").

4. `ArtifactState` frozen dataclass: kind (str), name (str), file_path (str), status (str), updated_at (str), error (str, default "")

5. `ModuleState` dataclass (mutable container): module_name (str), artifacts (list[ArtifactState], default_factory=list)
   - `transition(self, kind, name, file_path, new_status, error="") -> ModuleState` -- returns NEW ModuleState with updated artifact list (immutable pattern: filters out old entry for same kind+name, appends new ArtifactState with current UTC timestamp). Checks VALID_TRANSITIONS and logs warning if transition is invalid but does not block.

6. `save_state(state: ModuleState, module_path: Path) -> Path` -- writes JSON to module_path/.odoo-gen-state.json using dataclasses.asdict. Returns the state file path.

7. `load_state(module_path: Path) -> ModuleState | None` -- reads JSON from sidecar file, reconstructs ModuleState. Returns None if file missing or corrupted. Logs warning on parse errors.

8. `format_state_table(state: ModuleState) -> str` -- formats artifact states as human-readable table with status icons:
   - pending: "[ ]"
   - generated: "[G]"
   - validated: "[V]"
   - approved: "[A]"
   - unknown: "[?]"
   Sorted by kind. Includes ERROR line for artifacts with non-empty error field.

Use `logging.getLogger("odoo-gen.state")` for the logger.

Run tests -- they must ALL PASS.
