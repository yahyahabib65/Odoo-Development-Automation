  <action>
  1. Create `python/src/odoo_gen_utils/validation/data/error_patterns.json`:
     A library of 25 common Odoo error patterns. Each pattern has:
     - `id`: unique string identifier
     - `regex`: Python regex pattern to match against log text
     - `context_regex`: Optional additional regex for context confirmation (null if not needed)
     - `explanation`: Human-readable explanation of what went wrong
     - `suggestion`: Actionable fix suggestion
     - `severity`: "error" | "warning" | "info"

     Include these categories of patterns:

     **Model/Field Errors (8 patterns):**
     - field-not-found: KeyError for field names
     - model-not-found: Model does not exist
     - field-type-mismatch: Wrong field type in operation
     - missing-required-field: Required field not provided
     - duplicate-field: Field defined twice
     - invalid-field-definition: Malformed field declaration
     - missing-compute-method: Computed field references missing method
     - missing-inverse-method: Inverse method not found

     **XML/View Errors (5 patterns):**
     - xml-parse-error: ParseError / XMLSyntaxError
     - view-field-not-found: Field referenced in view not in model
     - invalid-view-arch: Invalid view architecture
     - duplicate-xml-id: Duplicate external ID
     - missing-action-reference: Action references missing model

     **Security Errors (3 patterns):**
     - access-denied: AccessError for model operations
     - missing-acl: No access rule for model
     - invalid-csv-format: Malformed ir.model.access.csv

     **Import/Dependency Errors (3 patterns):**
     - module-not-found: Module not installed / not found
     - import-error: Python import error
     - circular-dependency: Circular module dependency

     **Odoo 17 Deprecated API (4 patterns):**
     - deprecated-attrs: Using attrs= in XML views (use inline expressions)
     - deprecated-tree-tag: Using `<tree>` instead of `<list>` in views
     - deprecated-api-one: Using @api.one decorator (removed)
     - deprecated-openerp-import: Using `from openerp` instead of `from odoo`

     **Other Common (2 patterns):**
     - constraint-violation: SQL or Python constraint failed
     - database-error: PostgreSQL error during operations

  2. Create `python/src/odoo_gen_utils/validation/error_patterns.py`:

     - `load_error_patterns() -> tuple[dict, ...]`: Load and return patterns from the JSON data file. Cache after first load using a module-level variable.

     - `diagnose_errors(log_text: str, patterns: tuple[dict, ...] | None = None) -> tuple[str, ...]`:
       - If patterns is None, call load_error_patterns()
       - For each pattern, compile regex and search in log_text
       - If context_regex is not null, also confirm context match
       - For each match, format diagnosis string: "[severity] pattern.explanation\nSuggested fix: pattern.suggestion"
       - If no patterns match but traceback found: extract_traceback(log_text) and return ("Unrecognized error. Raw traceback:\n{traceback}",)
       - If no patterns match and no traceback: return empty tuple
       - Return tuple of diagnosis strings

  3. Update `python/src/odoo_gen_utils/validation/__init__.py` to export:
     ```python

     ```

     - Test each major category with realistic log snippets
     - Test multiple match scenario
     - Test unrecognized error fallback
     - Test empty input
